# UI Review + Cursor Handoff — TCDS

## Contexto del producto

Usuarios:
- comprador online (no técnico)
- quiere comprar rápido
- no quiere pensar demasiado

---

## Reglas UX clave

- claridad absoluta en:
  - stock express
  - encargo
  - personalización
- evitar ambigüedad
- evitar acciones implícitas
- mostrar tiempos SIEMPRE
- guiar la decisión de compra

---

## Objetivo

Analizar pantalla y mejorar:

- claridad
- jerarquía
- conversión
- comprensión de disponibilidad

---

## Qué buscar

- usuario no entiende si llega rápido o no
- falta de información de tiempos
- personalización confusa
- CTA poco claro
- jerarquía visual débil
- fricción innecesaria

---

## Output

1. Qué funciona  
2. Qué está mal  
3. Mejora prioritaria  
4. Prompt listo para Cursor  

---

## Reglas

- no reescribir toda la UI
- no cambiar diseño completo
- cambios puntuales de alto impacto
